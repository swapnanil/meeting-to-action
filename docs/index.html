How it works

Four steps.
Zero ambiguity.

Unstructured text in. Structured accountability out.

01
Input
Raw transcript file, stdin pipe, or API request — whatever fits your workflow
02
Extract
Claude identifies decisions, owners, deadlines, open questions, and risks
03
Validate
Pydantic models enforce a structured, consistent schema on every run
04
Output
Markdown report, JSON, or standalone follow-up email — your choice
Capabilities

Built for
accountability.

Every feature targets the gap between a meeting happening and work actually getting done.

01
Decisions vs Discussion
Distinguishes between things agreed upon and things still being debated — a distinction most manual note-takers get wrong
02
Owner Extraction
Pulls action item owners directly from natural language ("Sarah will handle...") — no structured input required
03
Risk Flags
Identifies blockers, unresolved dependencies, and concerns that could derail progress — each tagged with a severity level
04
Follow-up Email
Generates a complete, professional email ready to send with no editing. Addressed to "Team", signed as the meeting organiser
05
Pipe-friendly CLI
Works with stdin so it fits any shell workflow: cat notes.txt | python main.py run --stdin
06
Three Output Formats
JSON for programmatic use, Markdown for documentation, email-only mode for the follow-up draft
